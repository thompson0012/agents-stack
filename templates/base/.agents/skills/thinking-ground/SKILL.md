# Thinking Ground

Use this skill to improve the quality of reasoning before invoking `problem-definition` or `dynamic-problem-solving`.

This skill does **not** tell you whether anyone is truly in some privileged inner state. It gives a practical calibration protocol based on observable signals so the next round of reasoning is less distorted.

For a compact whole-system reminder, see `assets/system-pocket-card.md`.

## Core Contract

- Use observable signals, not metaphysical claims.
- Quote or paraphrase the decisive language or behavior that supports the state assessment.
- Treat motives, relief-seeking, and inner narratives as hypotheses only when the user stated them; otherwise leave them out.
- Do not romanticize uncertainty, depth, or "presence."
- Calibrate only enough to improve the next decision.
- Route quickly back to the right next tool: action, experiment, `problem-definition`, or `dynamic-problem-solving`.
- If the issue is better handled by direct experience, a real conversation, waiting, or human support, say so plainly.

## When to Trigger

Use this skill when one or more of these are present:

- the user already has an answer and wants validation
- urgency or fear is obviously shaping the reasoning
- the conversation keeps looping without updating
- the analysis sounds smart but not true
- the user is mixing identity defense with problem solving
- a previous framework output feels hollow, over-structured, or evasive
- the stakes are high enough that state quality matters

## Hard Limits

- This skill cannot verify what the user or model "really" feels internally.
- It cannot prove whether reasoning is authentic or performed.
- It is not therapy, diagnosis, or existential guidance.
- Its only honest success test is downstream: did the later reasoning and action improve?
- If you cannot describe the pressure without inventing an inner motive, stay at the level of observable urgency and route to the next action.

## State Scan

Use the minimum state labels necessary.

### State 1 — Defended

Reasoning is protecting a belief, decision, identity, or preferred answer.

Observable signals:

- the same conclusion is repeated despite counterevidence
- objections are explained away rather than incorporated
- one option is described in morally loaded or identity-loaded terms
- the user wants certainty more than understanding

### State 2 — Anxious

Reasoning is being used to escape uncertainty or fear.

Observable signals:

- urgency exceeds the real deadline
- catastrophic language outruns the evidence
- the user tries to force a conclusion quickly to stop discomfort
- reversible choices are being treated as irreversible

### State 3 — Performing

Reasoning is optimized for sounding intelligent, comprehensive, or impressive.

Observable signals:

- language gets more elaborate while the next action stays unclear
- "I don't know" is avoided even when it would be the honest answer
- complexity keeps growing but understanding does not
- analysis becomes theater for self-image or audience approval

### State 4 — Open Enough to Proceed

Use this as a working label, not a claim about inner purity.

Observable signals:

- the user can restate the strongest case against their view
- new evidence changes the shape of the answer
- uncertainty can be held without panic
- the next action becomes simpler, not more decorative

## Correction Protocols

Choose the active distortion. Do not run all protocols.

### For Defended reasoning — Assumption Strip

1. Name the answer being protected.
2. Ask: "What would have to be true for the opposite conclusion to be correct?"
3. Require the strongest opposing case in concrete terms.
4. Restate the problem in third person before proceeding.

### For Anxious reasoning — Deceleration

1. Name the feared outcome precisely.
2. Separate the real deadline from the emotional pressure. Describe the pressure in behavioral terms; do not infer why the user wants relief unless they already said it.
3. Ask: "If the feared outcome happened, what would we actually do next?"
4. Reclassify the choice as reversible or irreversible.

### For Performing reasoning — Simplicity Test

1. Restate the issue in plain language.
2. Remove anything that exists to sound deep rather than clarify.
3. Force one concrete next step.
4. If the simplified version still feels hollow, the framework is not the right next move.

### For Hollow output — Reality Return

1. Ask whether more analysis is truly needed.
2. If not, choose one of these exits:
   - talk to the relevant human
   - gather direct evidence
   - run a small experiment
   - wait for more information
3. If yes, route to the appropriate skill with a narrower problem statement.

## Routing Rules

After calibration, choose the next step explicitly.

- Problem unclear or solution-contaminated -> `problem-definition`
- Problem clear and complicated -> `dynamic-problem-solving`
- Situation complex -> small experiments, not deep analysis
- Situation chaotic -> stabilize first
- Framework boundary reached -> say that direct experience or human judgment is the correct next move

## Output Format

Return these sections:

### State Assessment
- defended, anxious, performing, or open enough to proceed

### Observable Signals
- 2-5 bullets tied to what was actually said or shown; quote or paraphrase the decisive language when possible

### Correction Applied
- the specific protocol used
- what changed after applying it, staying at the level of observable pressure rather than invented motive

### Next Step
- route to `problem-definition`
- route to `dynamic-problem-solving`
- gather evidence
- run an experiment
- stop analyzing and act

### Honesty Note
- one sentence stating the limit of the calibration

## Failure Modes to Avoid

- claiming to know the user's inner state
- smuggling motive attributions into a grounded-looking answer
- confusing poetic language with truth
- using the skill to delay action indefinitely
- treating calm tone as evidence of good reasoning
- treating self-awareness theater as actual update
- prescribing a ritual that the current context cannot support
