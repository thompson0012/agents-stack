# Output Contracts

Choose one contract before implementation. The contract is the truth about what the model may emit and what the host must validate.

## Decision Table

| Contract | Use when | Model emits | Host owns | Guardrails |
| --- | --- | --- | --- | --- |
| Standalone HTML document | Ephemeral demos, single-file explainers, sandboxed experiments | Full `html` document | iframe/container, message bridge, sanitization | Sandbox the render, keep dependencies explicit, provide at least one real interaction |
| Typed component schema | Production app integration, reusable agent surfaces, stable component libraries | JSON or structured props for allowlisted components | Validation, rendering, side effects, state wiring | Reject unknown components/props, keep actions typed, keep secrets and business logic host-side |
| Streamed UI | Long-running agent turns, progressive workflows, partial results | Ordered partial structure or component payloads | Skeletons, chunk assembly, cancellation, retries | Stable container IDs, partial-failure fallback, idempotent chunk handling |

## 1. Standalone HTML Document
Use this when the UI is disposable or needs to travel as a single artifact.

**Defaults**
- Emit a full document: `<html><head><body>`.
- Tailwind CDN is acceptable when no project design system is already wired.
- Chart.js CDN is fine for quick charts; prefer proven libraries over hallucinated SVG chart math.
- Render inside a sandboxed iframe or equivalent isolation boundary.

**Host-side rules**
- Strip markdown fences before injecting into `srcdoc`.
- Default to the smallest sandbox needed. Add privileges deliberately.
- Sanitize untrusted HTML if the model can emit arbitrary markup.
- Use a typed postMessage bridge for clicks or actions that must reach the host.

## 2. Typed Component Schema
Use this when the surface belongs inside a real product.

**Defaults**
- Define a closed component registry.
- Use typed validation (`zod` or equivalent) on every model payload.
- Keep the model output declarative: component name, props, children, action IDs.

**Host-side rules**
- Reject unknown component names.
- Reject extra props instead of ignoring them silently.
- Map action IDs to host-owned handlers.
- Keep data fetching, persistence, and privileged actions outside the model contract.

## 3. Streamed UI
Use this when the user should see useful structure before the whole answer is ready.

**Defaults**
- Send a skeleton immediately.
- Stream sections or components in stable order.
- Keep a fallback state for stream interruption or malformed chunks.

**Host-side rules**
- Preserve stable keys or container IDs across chunks.
- Make regeneration and retry explicit.
- Never let later chunks mutate unrelated surfaces implicitly.

## Vendor Notes
These are examples, not the contract:
- Standalone HTML can be generated by any model that returns text.
- Typed component schemas map naturally to app-owned renderers, CopilotKit-style actions, or JSON renderers.
- Streamed UI maps naturally to SDKs such as Vercel AI SDK, but the portability rule is the same: the stream protocol must stay host-owned and explicit.
