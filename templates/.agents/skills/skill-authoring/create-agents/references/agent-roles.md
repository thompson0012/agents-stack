# Agent Roles

## `task`
Use for one bounded coding task with clear inputs and a concrete file change.

## `explore`
Use for evidence gathering, codebase mapping, or comparing possible paths before implementation.

## `plan`
Use for turning a goal into durable scope, sequence, and risk boundaries without coding yet.

## `librarian`
Use for reusable knowledge, inventories, conventions, and lookups that should stay durable.

## Role rule
Keep each agent narrow. If one file would need to do two of these jobs, split the role.
