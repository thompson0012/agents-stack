# UI Component & States Reference

Domain-specific terms for text-to-design prompt augmentation.

## Interaction States

- **Default** — Resting state with no user interaction; neutral appearance
- **Hover** — Cursor-over feedback; subtle highlight or elevation change
- **Active / Pressed** — Mouse-down or touch-press state; depressed, darker feel
- **Focused** — Keyboard focus indicator; outline or ring for accessibility
- **Disabled** — Non-interactive, dimmed; conveys unavailable action
- **Loading / Skeleton** — Placeholder shimmer or pulse; content fetching indicator
- **Error** — Invalid or failed state; red accent, alert icon, inline message
- **Success** — Completed or confirmed state; green accent, check icon
- **Empty** — No data state; illustration + helpful copy + CTA prompt

## Navigation Types

- **Tab bar** — Horizontal tabs for peer-level views; top or bottom placement
- **Bottom nav** — Mobile primary navigation; 3–5 icons fixed at screen bottom
- **Side rail / Drawer** — Vertical icon rail or collapsible panel; desktop sidebar
- **Top nav** — Horizontal bar with logo, links, actions; universal header pattern
- **Mega menu** — Expanded multi-column dropdown; large site navigation
- **Breadcrumb** — Hierarchical path trail; wayfinding within deep structures
- **Pagination** — Numbered page controls; splitting long lists into pages

## Feedback Patterns

- **Toast / Snackbar** — Brief non-blocking message; auto-dismisses from bottom
- **Banner** — Persistent top-of-content alert; requires acknowledgment
- **Modal / Dialog** — Centered overlay blocking interaction; focused task or confirmation
- **Bottom sheet** — Slide-up panel from bottom; mobile-friendly, contextual actions
- **Inline alert** — Embedded message within content flow; contextual, non-blocking
- **Tooltip** — Small label on hover/focus; explains icon or truncated text
- **Popover** — Anchored floating panel; richer than tooltip, interactive content

## Form Elements

- **Input field** — Single-line text entry; label + field + validation states
- **Dropdown / Select** — Expandable list for single-choice selection from options
- **Checkbox** — Multi-select binary toggle; checked, unchecked, indeterminate
- **Radio** — Single-select within a group; mutually exclusive options
- **Toggle** — On/off switch; immediate effect, no confirmation needed
- **Slider** — Continuous range selector; min/max with step granularity
- **Date picker** — Calendar or input for date selection; single, range, or multiple
- **Search** — Text input with icon for querying; autocomplete, recent, suggestions
