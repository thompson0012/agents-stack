# Dynamic Problem Solving — Quick Invoke

Use this when you need the shortest faithful version of the workflow.

## Entry check

1. Can the problem be stated in 1-2 sentences?
   - If no -> use `using-reasoning/problem-definition`.
2. Is this a complicated problem rather than simple, complex, or chaotic?
   - If no -> do not force this skill.

## Minimal flow

1. Record first instinct and emotion.
2. Classify type, stakes, time tier, uncertainty, and core tension.
3. Pick 2-4 lenses only.
4. Apply each lens independently.
5. Force collision:
   - where do they agree?
   - where do they contradict?
   - what is still outside the frame?
6. End with:
   - recommendation
   - tradeoff accepted
   - most dangerous assumption
   - first test
   - what would prove the analysis wrong

## Non-negotiables

- one human lens + one structural lens
- never exceed 4 lenses
- disagreement is signal
- action beats elegant summary
